# 算法复杂度

## 核心概念

复杂度用于描述算法随着输入规模增长时，时间或空间消耗的增长趋势。

常见写法是大 O 表示法，例如 `O(1)`、`O(n)`、`O(n log n)`、`O(n^2)`。

## O(1)

`O(1)` 表示常数时间，操作次数不随输入规模增长。

```text
访问数组下标 arr[0]
```

无论数组多长，访问一个指定位置通常都是常数时间。

## O(n)

`O(n)` 表示线性时间，输入越大，操作次数大致按比例增长。

```js
for (const item of list) {
  console.log(item)
}
```

遍历数组通常是 `O(n)`。

## O(n^2)

双重循环常见复杂度是 `O(n^2)`。

```js
for (let i = 0; i < n; i++) {
  for (let j = 0; j < n; j++) {
    // ...
  }
}
```

当 n 变大时，`O(n^2)` 增长很快。

## O(log n)

`O(log n)` 常见于每次把问题规模减半的算法，例如二分查找。

```text
每次只保留一半搜索范围
```

## O(n log n)

很多高效排序算法是 `O(n log n)`，例如归并排序和快速排序的平均情况。

## 空间复杂度

空间复杂度描述算法额外使用的内存。

如果算法新建了一个和输入规模相同的数组，空间复杂度通常是 `O(n)`。

## 常见误区

第一个误区是只看代码行数，不看循环结构。

第二个误区是忽略嵌套循环。两层循环不一定永远是 `O(n^2)`，还要看每层循环的范围。

第三个误区是只关注时间复杂度，不关注空间复杂度。很多算法是在时间和空间之间做取舍。

## 面试表达

回答复杂度时，可以按顺序说：输入规模是什么、主要循环或递归是什么、时间复杂度是多少、额外空间是多少。
