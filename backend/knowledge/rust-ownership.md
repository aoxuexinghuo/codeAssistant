# Rust 所有权

## 核心概念

所有权是 Rust 保证内存安全的核心机制。每个值都有一个所有者，当所有者离开作用域时，这个值会被自动释放。

Rust 不依赖垃圾回收器，而是通过所有权、借用和生命周期在编译期检查内存安全。

## 三条基本规则

第一，每个值都有一个所有者。

第二，同一时间一个值只能有一个所有者。

第三，当所有者离开作用域，值会被释放。

## 所有权转移

```rust
let s1 = String::from("hello");
let s2 = s1;
```

这段代码中，`String` 的所有权从 `s1` 转移到 `s2`。转移之后，`s1` 不能再使用。

## Copy 类型

```rust
let x = 1;
let y = x;
```

整数、布尔值、字符等简单类型通常实现了 `Copy`，赋值时会复制值，所以 `x` 仍然可以继续使用。

## 借用

借用允许函数或变量临时访问一个值，但不拿走所有权。

```rust
fn print_len(s: &String) {
    println!("{}", s.len());
}
```

这里 `&String` 表示不可变借用。

## 可变借用

```rust
fn add_text(s: &mut String) {
    s.push_str("!");
}
```

可变借用允许修改值，但同一时间只能有一个可变借用。

## 借用规则

同一时间可以有多个不可变借用，或者一个可变借用，但不能两者同时存在。

这条规则可以避免数据竞争。

## 常见误区

第一个误区是把所有权转移理解成“复制”。对于 `String` 这类堆上数据，赋值通常是移动所有权，而不是复制完整数据。

第二个误区是只看变量名，不看函数参数。判断是否 move，关键要看参数是 `T`、`&T` 还是 `&mut T`。

第三个误区是认为借用检查器只是限制。实际上它是在编译期帮你发现悬垂引用和数据竞争。

## 学习建议

读 Rust 代码时，先判断值是否发生 move，再判断是否发生 borrow，最后看 borrow 的作用域在哪里结束。
